# Security Rules

## Credentials
- НИКОГДА не хардкодить пароли, токены, webhook URL в коде
- Все секреты — через переменные окружения и pydantic-settings
- `.env.prod` защищён хуком от случайной записи через Claude

## Логирование
- НЕ логировать: пароли, токены, webhook URL, полные authorization headers
- Маскировать чувствительные данные: первые 3 символа + `***`

## SSL
- Camunda: `verify=False` через `ssl_patch.py` (известное ограничение библиотеки)
- Для всех остальных HTTP-запросов: `verify=True` по умолчанию
- НЕ отключать SSL верификацию без явной причины

## Файлы, которые нельзя коммитить
- `.env.prod`, `.env.dev` — содержат секреты
- `*.pem`, `*.key` — сертификаты
- `logs/` — могут содержать чувствительные данные
